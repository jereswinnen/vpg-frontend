# Workflow: Meta Description

<required_reading>
**Lees deze referentiebestanden NU:**
1. references/vpg-approved-content.md (primair - goedgekeurde teksten en USPs)
2. references/cp-trappen.md (secundair - tone-of-voice inspiratie)
</required_reading>

<process>

<step_1>
**Verzamel informatie via vragen**

Stel deze vragen aan de gebruiker met AskUserQuestion:

1. "Voor welke pagina?" - Opties:
   - Homepage
   - Over ons
   - Contact
   - Realisaties/Portfolio
   - Productpagina (welke?)

2. "Primaire zoekwoorden?" - Vrije invoer
   Voorbeelden: "trappen op maat", "houten trap België", "trappenmaker"

3. "Wat moet de lezer doen?" - Opties:
   - Meer informatie zoeken (doorklikken)
   - Contact opnemen
   - Inspiratie opdoen
   - Offerte aanvragen
</step_1>

<step_2>
**Schrijf meta description**

Regels:
- **Lengte:** 150-160 karakters (Google kapt af bij ~160)
- **Begin met kernboodschap** (eerste 60 tekens tellen meest)
- **Bevat zoekwoord** (natuurlijk verwerkt)
- **Eindigt met actie of belofte**
- **Geen dubbele aanhalingstekens** (breekt SERP weer)

Structuur:
[Wat bieden we] + [Onderscheidend element] + [Actie/belofte]
</step_2>

<step_3>
**Genereer 3 varianten**

Bied de gebruiker 3 opties:
1. Focus op maatwerk
2. Focus op Belgisch/lokaal
3. Focus op persoonlijke aanpak

Laat gebruiker kiezen of combineren.
</step_3>

<step_4>
**Valideer**

Check:
- [ ] Exact karakteraantal (toon dit)
- [ ] Zoekwoord aanwezig
- [ ] Geen dubbele aanhalingstekens
- [ ] Actieve, uitnodigende toon
- [ ] Geen loze beloftes
</step_4>

</process>

<success_criteria>
Meta description is compleet wanneer:
- [ ] 150-160 karakters
- [ ] Zoekwoord natuurlijk verwerkt
- [ ] Duidelijke waardepropositie
- [ ] Actie-gerichte afsluiting
- [ ] Gebruiker heeft gekozen uit varianten
</success_criteria>
